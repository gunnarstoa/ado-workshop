Lab A: Automate Your ADO Job

Create workflow templates to schedule your ADO links automatically
Lab A ⏱ 60 min Hands-On
Meridian Corp Scenario

You are an Anaplan partner who has just completed building Meridian Corp's ADO pipelines during the development phase. The HR team needs employee payroll data refreshed nightly at 11 PM UTC. The product team needs product inventory data refreshed weekly on Sunday mornings. Your task is to automate both ADO jobs using Workflow.

What You'll Need

  • Your ADO workspace from the Academy course (with the HR model ADO job and FPA model ADO job)
  • Workspace Admin access
  • Access to Workflow in your Anaplan platform
No Academy models?

If you don't have models from the Academy course, use any ADO job you've built. The steps are the same — just substitute your model and link names where the lab references HR Model or FPA Model.

1
Create Your First Workflow Template — Employee Data
⏱ 15 min
Objective Create a Workflow template named "ADO | Employee Data | Dev | Nightly" and add your HR Model ADO link as the first step.
  1. 1
    From the Anaplan platform, click the Workflow icon in the left navigation bar (it looks like a set of connected nodes or is labeled "Workflow" depending on your version).
  2. 2
    Click New Template in the top-right corner of the Workflow screen.
  3. 3
    In the Name field, enter: ADO | Employee Data | Dev | Nightly
  4. 4
    In the Description field, enter: Automates nightly HR payroll data load for Meridian Corp Dev environment
  5. 5
    Click Add Step. Select ADO Action as the step type.
  6. 6
    In the ADO Action dropdown, find and select your HR Model ADO link (the link that loads employee/payroll data into the HR model from the Academy course).
  7. 7
    Click Save to save the template (do not configure the schedule yet — that is Mini-Lab 2).

✅ Done when:
  • A template named "ADO | Employee Data | Dev | Nightly" exists in your Workflow list

  • Status shows "Saved" (no schedule yet)
2
Configure the Schedule
⏱ 10 min
Objective Set a nightly schedule on the employee data template so ADO runs automatically at 23:00 UTC every day.
  1. 1
    Open the template you created in Mini-Lab 1: ADO | Employee Data | Dev | Nightly.
  2. 2
    Click the Schedule tab (or find the Schedule section on the right panel).
  3. 3
    Set Frequency to Daily.
  4. 4
    Set Time to 23:00 (UTC). Note that Anaplan schedules run in UTC — confirm this matches the customer's requirement.
  5. 5
    Set Start date to today's date (or the desired go-live date).
  6. 6
    Toggle the Enabled switch to ON.
  7. 7
    Click Save Schedule.
📸

✅ Done when:
  • The template shows "Scheduled" status
  • Next run time is displayed (approximately 23:00 UTC tonight or the next occurrence)
3
Create a Second Template — Product Data
⏱ 15 min
Objective Create a second, separate Workflow template for product data and configure a weekly schedule. Understand why separate templates matter.
Why a Separate Template?

Product master data changes slowly compared to employee data. If you combined both in one template and the employee data load failed, the product data load would also be blocked or skipped. Separate templates give each dataset its own run history, its own schedule, and its own failure scope.

  1. 1
    Return to the Workflow templates list. Click New Template.
  2. 2
    Name it: ADO | Product Master | Dev | Weekly
  3. 3
    Add an ADO Action step. Select your FPA Model ADO link (the link that loads product or financial data from the Academy course).
  4. 4
    Go to the Schedule tab. Set Frequency to Weekly.
  5. 5
    Set the day to Sunday and time to 06:00 UTC.
  6. 6
    Enable the schedule and save.
✅ Done when:
  • A second template "ADO | Product Master | Dev | Weekly" exists in your Workflow list
  • It shows "Scheduled" with next run on the upcoming Sunday at 06:00 UTC
  • You now have two separate templates, each with their own schedule
4
Verify and Test the Schedule
⏱ 10 min
Objective Trigger a manual run of your employee data template to verify the pipeline executes correctly before relying on the automated schedule.
  1. 1
    Open the ADO | Employee Data | Dev | Nightly template.
  2. 2
    Click Run Now (or "Test Run" depending on your version). This triggers the ADO link immediately, bypassing the schedule.
  3. 3
    Watch the Run History tab. Wait for the status to change from "Running" to "Completed" or "Failed".
  4. 4
    If Completed: navigate to the HR Model in Anaplan and verify that data has been updated (check the last-loaded timestamp or a data value you know should have changed).
  5. 5
    If Failed: note the error message in Run History, then check the Troubleshooting page for guidance.
