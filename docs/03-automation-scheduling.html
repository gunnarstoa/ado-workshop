Automating ADO Jobs

Using Anaplan Workflow to schedule and automate ADO runs
Automation (Workflow) ⏱ 30 min read

Why Automate?

When you build ADO in the Academy course, you run it manually — you click "Run" to trigger the pipeline. In production, that's not viable. Customers need data refreshed on a schedule: HR data nightly, financial data weekly, inventory data twice daily. Workflow is how you get there.

Workflow is Anaplan's built-in process automation tool. It allows you to create reusable templates that execute one or more ADO actions (links) on a defined schedule — no manual triggering required, no external scheduler needed.

Pricing Note

Workflow is included in Anaplan's V5 pricing for both Advanced and Standard tiers. Essentially all customers can use Workflow — you do not need a separate purchase or add-on.

How Workflow Works with ADO

At its simplest, a Workflow template for ADO contains one step: run an ADO link. That link triggers the ADO pipeline — source → transform → load. The template then runs on a schedule you define.

Tell Workflow Template A container that holds one or more Workflow steps. For ADO, each step maps to an ADO link (the run action).
Show ADO Link as Step The Workflow step references the ADO link. When the step runs, it triggers the full ADO pipeline behind that link.
Do Schedule It Set a recurring schedule (daily, weekly, custom cron) on the template. Anaplan runs it automatically.

4 Steps to Create a Workflow Template for ADO

1

Navigate to Workflow

In the Anaplan platform, go to the workspace and open Workflow from the left navigation. You will see a list of existing workflow templates (empty if this is a new workspace).

2

Create a New Template

Click New Template. Give the template a clear, descriptive name. Use a naming convention that identifies the dataset and environment, for example:

  • ADO | Employee Data | Dev | Nightly
  • ADO | Product Master | Dev | Weekly
  • ADO | HR Payroll | Prod | Nightly
📸
[Screenshot Placeholder]
New Template dialog — name field highlighted, showing the naming convention example.
3

Add the ADO Link as a Step

Inside the template, click Add Step and select ADO Action (or "Run ADO Link" depending on your platform version). From the dropdown, select the ADO link you want to automate. This is the link you built in the Academy course that pushes data into your Anaplan model.

Note

Only links (not transformation views or source data) can be added as Workflow steps. The link encapsulates the full pipeline — when Workflow runs the link, ADO automatically runs the transformation chain and loads the result.

📸
[Screenshot Placeholder]
Add Step dialog — ADO Action type selected, link dropdown showing available ADO links in the workspace.
4

Set the Schedule

Once the step is configured, go to the Schedule tab of the template. Configure:

  • Frequency: Daily, Weekly, Monthly, or Custom (cron expression)
  • Time: The time the run should trigger (UTC-based — remember to account for your customer's local timezone)
  • Start date: When the schedule becomes active
  • Enabled: Toggle on to activate the schedule

Best Practice: Separate Templates Per Dataset Type

Best Practice

Create one Workflow template per dataset type, not one template that runs everything. Employee data and product data have different schedules, different failure modes, and different business owners. If they share a template and one fails, both are blocked.

Here's why separate templates matter in practice:

Scenario Single Template (bad) Separate Templates (good)
Employee data fails due to HRIS export issue Product data also blocked or skipped Product data runs independently, unaffected
HR team needs to change payroll run time Must coordinate with FPA team who shares the template HR template schedule updated independently
Troubleshooting which dataset caused a load failure Must inspect combined run log Each template has its own run history and logs
Different datasets promoted to production at different times Template must be duplicated and partially updated Each template is promoted independently on its own timeline

Meridian Corp Example

For the Meridian Corp implementation, two workflow templates are needed from day one:

Template 1: Employee Payroll

  • Name: ADO | Employee Data | Dev | Nightly
  • Step: HR Payroll ADO Link (Dev)
  • Schedule: Daily at 23:00 UTC
  • Rationale: Payroll data must be current before the morning planning session (Pacific time)

Template 2: Product Inventory

  • Name: ADO | Product Master | Dev | Weekly
  • Step: Product Master ADO Link (Dev)
  • Schedule: Weekly on Sunday at 06:00 UTC
  • Rationale: Product master changes slowly; weekly refresh before Monday planning is sufficient
Timezone Tip

Anaplan Workflow schedules run in UTC. Always confirm the customer's local timezone and convert accordingly. A customer in Japan (JST = UTC+9) wanting an 8 AM local run should schedule their Workflow at 23:00 UTC the previous day.
